val temperature: Double? = null

The sampling temperature, between 0 and 1. Higher values like 0.8 will make the output more random, while lower values like 0.2 will make it more focused and deterministic. If set to 0, the model will use log probability to automatically increase the temperature until certain thresholds are hit.

timestampGranularities
Link copied to clipboard
val timestampGranularities: List<TimestampGranularity>? = null

The timestamp granularities to populate for this transcription. responseFormat must be set AudioResponseFormat.VerboseJson to use timestamp granularities. Either or both of these options are supported: TimestampGranularity.Word, or TimestampGranularity.Segment. Note: There is no additional latency for segment timestamps, but generating word timestamps incurs additional latency.
